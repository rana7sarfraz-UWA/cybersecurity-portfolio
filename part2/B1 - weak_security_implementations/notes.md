B1 — Weak Security Implementations

Five weak security implementations were identified and analysed from common real-world systems and applications.
The examples included weak passwords, lack of multi-factor authentication, exposed configuration files, world-readable permissions, and reuse of default credentials.
These weak implementations increase the risk of unauthorised access, credential theft, privilege escalation, and data exposure.
Many of these vulnerabilities occur due to poor security configuration or failure to follow secure system hardening practices.
The activity demonstrated how small security misconfigurations can significantly weaken the overall security posture of a system.
