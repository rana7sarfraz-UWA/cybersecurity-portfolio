B24 — Access Control Implementation

An access control system was implemented using Linux file permissions and ACLs to restrict file access between different users.
Two or more user roles were created and different permission levels were assigned to simulate real-world multi-user security.
Permissions were configured using chmod and setfacl so that only authorised users could read or modify specific files, while others were denied access.
This demonstrated how fine-grained access control can enforce security policies at the file level.
The implementation showed that proper permission management can effectively enforce the principle of least privilege and prevent unauthorised data access between users.
